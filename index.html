⬡ Toolchain — Phase 1
Toolchain: Vite + ESLint + Prettier
Без изменения UI — только сборщик и линтер. Нулевой регресс.
hot reload CI ready нулевой риск
vite.config.js
entry + legacy plugin
Заменяет ручное подключение скриптов. Один HTML-entrypoint, tree-shaking, hot module replacement в dev-режиме.
// vite.config.js
export default {
  build: { rollupOptions: {
    input: 'src/main.js'
  }},
  plugins: [legacy()]
}
ESLint no-var rule
запрет var, only let/const
Все var в коде станут ошибкой CI. Только let и const — никаких неожиданных hoisting-багов.
// .eslintrc
{ "rules": {
  "no-var": "error",
  "prefer-const": "warn"
}}
CI: build + lint
GitHub Actions / GitLab CI
Каждый push проходит vite build + eslint. Merge-блок при ошибке. Prettier — автоформат при коммите через husky.
# .github/workflows/ci.yml
steps:
  - run: npm ci
  - run: npm run lint
  - run: npm run build

· UI не меняется — только сборка
· Никакого рефакторинга логики
· Fallback: legacy() плагин для старых браузеров
· Prettier не трогает семантику — только форматирование

Карта модулей
user.js
G.uid · G.name
Идентификация пользователя. G.uid и G.name становятся явными экспортами.

⬡ Strangler Fig — Phase 3
Strangler Fig Pattern
Мартин Фаулер · не переписывать всё — заменять по одному острову
Как фиговое дерево-душитель обвивает старое дерево снаружи, пока то не исчезнет — React-компоненты обволакивают HTML-монолит, постепенно вытесняя его.
нулевой простой обратимо по одному блоку
Принцип работы
A
Выбрать самый изолированный блок
Leaderboard row · Balance card · Result screen
B
Написать React-компонент рядом
Монолит ещё работает — компонент готовится
C
Вмонтировать через createRoot
React рендерит в div-контейнер внутри HTML
D
Удалить старый HTML-блок
Монолит стал меньше. Повторить с следующим.
Порядок миграции — от изолированного к сложному
1
LeaderboardRow.jsx
начать здесь
Чистая функция: принимает player, рендерит строку. Нет side effects. Идеальный первый остров.
// LeaderboardRow.jsx
export const LeaderboardRow = ({ player, rank }) => (
  <div className="lb-row">
    <span className="lb-rank">{rank}</span>
    <span>{player.name}</span>
  </div>
);
2
BalanceCard.jsx
Подписывается на balance.js (Phase 2). Реактивное отображение баланса и P&L без прямого доступа к G.
// BalanceCard.jsx
import { getBalance } from './balance';
export const BalanceCard = () => {
  const [bal, setBal] = useState(getBalance());
  // …useEffect sync
};
3
ResultScreen.jsx
Экран результата раунда — самодостаточный, получает данные из game.js. Заменяет сложный HTML-шаблон.
// main.js — монтируем остров
import { createRoot } from 'react-dom/client';
import { ResultScreen } from './ResultScreen';
createRoot($('result-mount'))
  .render(<ResultScreen />);
4
App.jsx
финал
Когда все острова заменены — монолитный HTML сводится к одному <div id="root">. Strangler fig завершён.
⬡ ПРАВИЛО ФАУЛЕРА
· Никогда не трогать больше одного «острова» за раз
· Старый код остаётся рабочим до момента удаления
· Если компонент сложнее оригинала — стоп, пересмотреть
· Feature flag на каждый остров: можно откатить за секунду

⬡ State Management — Phase 4
State Management: Zustand или Jotai
Атомарный стейт вместо G-объекта. Devtools. Time-travel debug.

Когда компонентов уже несколько, var G как шина данных заменяется на Zustand (минимальный boilerplate) или Jotai (атомарная реактивность). Server state — отдельно, через React Query.

Zustand Jotai React Query Devtools
Zustand
минимальный boilerplate
Один create() вызов вместо var G. Компоненты подписываются на нужные поля — перерисовка только при их изменении. Встроенные Devtools и middleware для persist.
// store/gameStore.js
import { create } from 'zustand';

export const useGameStore = create((set) => ({
  bal: 250, stake: 25, rounds: 0,
  addFunds: (n) => set(s => ({ bal: s.bal + n })),
  setStake: (n) => set({ stake: n }),
}));
Jotai
атомы, реактивно
Каждое поле — отдельный атом. Компонент подписывается только на то, что ему нужно. Производные атомы (derived) вычисляются автоматически — идеально для trustRate, pnl.
// atoms/game.js
import { atom } from 'jotai';

export const balAtom = atom(250);
export const roundsAtom = atom(0);
export const trustsAtom = atom(0);
// derived — вычисляется автоматически
export const trustRateAtom = atom(
  (get) => get(roundsAtom)
    ? Math.round(get(trustsAtom)/get(roundsAtom)*100)
    : 0
);
React Query
server state отдельно
Server state (глобальная статистика, лидерборд) — не в Zustand/Jotai, а в React Query. Автокэш, фоновый рефетч, stale-while-revalidate. Заменяет ручной setInterval для G.global.
// вместо setInterval каждые 3с
const { data } = useQuery({
  queryKey: ['globalStats'],
  queryFn: fetchGlobalStats,
  refetchInterval: 3000,
  staleTime: 1000,
});
До → После
ДО
var G = { bal: 250 };
G.bal += n;
// все читают G напрямую
// нет реактивности
// ручной updateAll()
ПОСЛЕ
const bal = useGameStore(
  s => s.bal
);
// авторендер при изменении
// Devtools + persist
⚡ DEVTOOLS + TIME-TRAVEL
· Zustand DevTools — состояние в Redux DevTools прямо в браузере
· Jotai DevTools — атомы, их значения и зависимости в реальном времени
· temporal middleware — time-travel: перемотка состояния назад
· persist middleware — автосохранение в localStorage без ручного кода
✓ КОГДА ВЫБИРАТЬ
· Zustand — если нужен простой store с action-методами (как Redux, но без шаблонного кода)
· Jotai — если стейт гранулярный и производные значения важны (trust rate, pnl вычисляются автоматически)
· React Query — для всего, что приходит с сервера: никогда не смешивать с клиентским стейтом
⬡ Тесты + деплой — Phase 5
⚡ ЗАЧЕМ СЕЙЧАС
Сейчас в проекте нет ни одного теста и нет preview-деплоя на PR. Любая правка — слепой полёт: сломал ли ты баланс, matchmaking или UI — узнаёшь только руками. Phase 5 закрывает этот пробел: Unit → Integration → E2E и автоматический preview на каждый пул-реквест.
Пирамида тестов
Vitest
unit · быстро
Тестирует чистую логику без DOM. Запускается за <1 с. Идеально для бизнес-правил Verdict.
// fmt, rake, resolve
test('rake 5% only on win', () => {
  expect(calcRake(100, 'win')).toBe(5)
  expect(calcRake(100, 'loss')).toBe(0)
})
Testing Library
component integration
Монтирует React-компонент (после Phase 3) и кликает кнопки как пользователь. Никаких моков внутренностей — только видимое поведение.
test('Trust btn updates balance', () => {
  render(<DecisionCard />)
  fireEvent.click(screen.getByText('Trust'))
  expect(screen.getByTestId('balance'))
    .toHaveTextContent('$274')
})
Playwright
E2E · реальный браузер
Открывает Chromium/Firefox/Safari, проходит полный раунд от входа до результата. Ловит то, что unit-тесты пропускают: routing, анимации, BroadcastChannel.
test('full round demo mode', async ({ page }) => {
  await page.goto('/')
  await page.click('text=Demo')
  await page.click('text=Trust')
  await expect(page.locator('.result'))
    .toBeVisible()
})
Preview деплой
Vercel / Netlify Preview
PR → URL за 60 с
Каждый PR автоматически получает уникальный URL вида verdict-pr-42.vercel.app. Ревьюер открывает — и сразу живое приложение, без локального запуска. Это «нормальное деплоирование», которого сейчас нет.
Docker + feature flags
production gate
Docker фиксирует окружение между preview и prod. Feature flags (LaunchDarkly / own impl) позволяют выкатывать real-money логику на 1% пользователей, не разворачивая отдельный деплой.
